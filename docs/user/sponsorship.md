# 赞助权益体系

> ⚠️ **声明**：本规则最终解释权归 RelayPulse 所有。规则可能会根据实际运营情况进行调整，变更时将提前通知。

---

## 1. 术语定义

### 1.1 角色说明

| 角色 | 定位 | 准入条件 |
|------|------|----------|
| **普通访客** | 未赞助的网站访问者 | 无 |
| **用户赞助** | 个人赞助者，可提交通道，享受 Aff 权益 | 完成赞助 |
| **服务商赞助** | API 服务商，需接受监督 | 完成赞助 |
| **一级赞助商** | 商业赞助入门级 | 收录监测满 **7 天**（累计），名额不限 |
| **二级赞助商** | 商业赞助进阶级 | 收录监测满 **15 天**（累计），限 5 个名额 |
| **顶级赞助商** | 商业赞助最高级 | 收录监测满 **30 天**（累计），限 2 个名额 |
| **RelayPulse** | 平台方，提供监测服务与协调支持 | — |

### 1.2 通道类型

- **收录通道**：被 RelayPulse 收录监测的 API 通道，上限 100 个
- **徽标通道**：可展示赞助商专属徽标的通道，按赞助档位限制数量

### 1.3 符号约定

| 符号 | 含义 |
|------|------|
| ✅ | 支持 / 有此权益 |
| ❌ | 不支持 / 无此权益 |
| — | 不适用 |

## 3. 分角色说明

### 3.1 用户赞助

**权益**：
- 申请检测通道收录（上限 100 个）
- 配置 Aff 推广链接
- 享受 5 分钟监测频率

**限制**：
- 无专属徽标
- 不适用置顶排名

**适用场景**：个人用户希望推荐好用的 API 服务商，同时获得推广收益。

### 3.2 服务商赞助

**权益**：
- 申请检测通道收录
- 享受 4 分钟监测频率

**义务**：
- 接受用户反馈
- 配合平台处理问题
- 接受负向风险徽标（问题成立时）

**适用场景**：API 服务商希望通过透明监测展示服务质量。

### 3.3 一级 / 二级 / 顶级赞助商

| 等级 | 准入条件 | 监测频率 | 徽标通道数 | 定制 Tips | 名额上限 |
|------|----------|----------|------------|-----------|----------|
| 一级 | 收录监测满 7 天 | 3m | 1 | ❌ | 不限 |
| 二级 | 收录监测满 15 天 | 2.5m | 5 | ✅ | 5 个 |
| 顶级 | 收录监测满 30 天 | 2m | 不限* | ✅ | 2 个 |

**权益**（递进关系）：
- 专属赞助徽标
- 适用置顶排名规则
- 更高监测频率
- 更多徽标通道配额

**义务**（与服务商相同）：
- 接受用户反馈
- 配合平台处理问题
- 接受负向风险徽标

### 3.4 平台方承诺

RelayPulse 作为平台方承诺：

1. **数据真实性**：保证不篡改监测数据
2. **中立协调**：反馈处理中保持中立，提供沟通支持
3. **规则透明**：规则变更提前公示通知
4. **隐私保护**：不泄露赞助商敏感信息

## 6. 商业展示规则

### 6.1 置顶排名

适用于一级及以上赞助商，需同时满足：
- 无风险徽标
- 可用率 ≥ 95%（可配置）

**排序规则**：
1. 按赞助级别降序：顶级 > 二级 > 一级
2. 同级别按可用率降序
3. 同一服务商同一服务类型最多置顶 1 个

**置顶时机**：
- 仅页面初始加载时生效
- 用户点击排序按钮后，按用户选择排序

### 6.2 徽标与 Tips 规范

**徽标**：
- 一级及以上赞助商可展示专属徽标

**定制 Tips**：
- 二级及以上可自定义徽标提示文字
- 字数限制：20 字以内
- 禁止虚假宣传、攻击竞争对手、违法违规内容

### 6.3 Aff 规范

- 仅用户赞助可配置 Aff 链接
- Aff 链接必须指向服务商官网
